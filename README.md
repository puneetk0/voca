## What it does

- **Describe a form in plain language** → AI drafts the questions, a matching conversation personality, and any branching logic your prompt implies.
- **Share one link** → respondents answer by **voice or text**, on any phone or laptop. No app, no login. The mic listens until they tap — no cut-offs mid-thought.
- **A warm AI voice interviews them** one question at a time: understands corrections mid-sentence, handles Hinglish code-switching, confirms what it heard, and extracts answers **verbatim** (fillers removed, wording preserved — never paraphrased).
- **Conditional branching (decision-tree forms)** — any choice question can route to different follow-ups or end the form early ("Not interested? → Why not? → done"). Routing is computed in code, never left to the LLM; authored visually in the editor or generated by the AI.
- **Team roles** — invite moderators (edit forms, manage settings) and viewers (read-only results/analytics) by email. Invitees without an account set a password right from the invite link; access is enforced end-to-end with Postgres RLS.
- **Real-time dashboard** with per-field breakdowns, drop-off funnel, completion time, device/browser split, answer sentiment, audio playback, and CSV export.
- Per-form **AI personality** (context, tone, welcome message, language), **post-submit redirect**, **email notifications**, **admin preview mode**, and a **health check**.

## Tech stack

| Layer | Choice |
| --- | --- |
| Framework | **Next.js 16** (App Router) · React 19 · TypeScript · Tailwind v4 |
| Data / auth / storage / realtime | **Supabase** (Postgres + RLS, email/password auth, Storage, Realtime) |
| LLM (conversation + form generation) | **Groq** `llama-3.3-70b` → **Cerebras** `gpt-oss-120b` fallback |
| Speech-to-text | **Sarvam Saarika** (Hinglish/Indian-name native) → **Groq Whisper** fallback |
| Text-to-speech | **Sarvam bulbul:v3** → **Groq Orpheus** (English) → browser speech → captions mode |
| Rate limiting | Upstash Redis (in-memory fallback built in) · Email: Resend · Analytics: PostHog |

There is **no Google Cloud dependency.** At least one of `GROQ_KEY` / `CEREBRAS_API_KEY` is required; everything else is optional and degrades gracefully.

## Local setup

```bash
git clone https://github.com/puneetk0/voca.git
cd voca
npm install          # Node.js >= 20.9
cp .env.example .env.local   # then fill in the values
npm run dev
```

**1. Environment** — see [`.env.example`](.env.example). Minimum to run: Supabase URL/keys + one LLM key (`GROQ_KEY` or `CEREBRAS_API_KEY`). Add `SARVAM_API_KEY` for real voice.

**2. Database** — in the Supabase SQL editor, run the migrations **in order**:

```
supabase/migrations/0001_reconcile_and_sessions.sql
supabase/migrations/0002_form_personality.sql
supabase/migrations/0003_storage_waitlist_realtime.sql
supabase/migrations/0004_team_roles.sql
```

They are idempotent. `0003` also creates the public storage buckets (`audio_submissions`, `user_files`), the waitlist insert policy, and enables realtime on `responses`. `0004` adds `form_members` / `form_invites` and the role-based RLS policies. (For a brand-new project you can instead run [`supabase/schema.sql`](supabase/schema.sql), then `0003`'s bucket/realtime section.)

**3. Run** — open `http://localhost:3000`, sign in, and create your first form. Use the **Preview** button on the form dashboard to test it without saving responses.

## Production deploy (checklist)

Most of these fail **silently** if skipped:

- [ ] Run migrations `0001` → `0004` on the production database.
- [ ] Confirm buckets `audio_submissions` + `user_files` exist and are **public** (0003 creates them).
- [ ] Set env vars on the host (Vercel): `NEXT_PUBLIC_SUPABASE_URL/ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `GROQ_KEY` (+ `GROQ_KEY_2/3`), `CEREBRAS_API_KEY`, `SARVAM_API_KEY` (+ optional `SARVAM_SPEAKER`, `GROQ_TTS_VOICE`), `NEXT_PUBLIC_APP_URL`, `ALLOWED_ADMIN_EMAILS`, `UPSTASH_REDIS_REST_URL/TOKEN` (enables distributed rate limiting), `NEXT_PUBLIC_POSTHOG_KEY/HOST`, `RESEND_API_KEY` + `EMAIL_FROM` (+ optional `EMAIL_REPLY_TO`, `WAITLIST_FORM_URL`).
- [ ] For email: verify your domain in Resend, set `EMAIL_FROM` to an address on it, and add a **DMARC** TXT record (`v=DMARC1; p=none; rua=mailto:you@…`) — new domains land in spam without it.
- [ ] Accept the **Groq Orpheus** terms once (console.groq.com playground → `canopylabs/orpheus-v1-english`) so English TTS has a real fallback voice when Sarvam blips.
- [ ] `ALLOWED_ADMIN_EMAILS` **must** be set in production — an empty value denies all admin access by design. Invited team members bypass the allowlist via their `form_members` row.
- [ ] Enable **Vercel Skew Protection** (Settings → Advanced) so open tabs survive new deployments without 404s.
- [ ] In Supabase → Auth → URL Configuration, set the Site URL and redirect URLs to your production domain.

The server logs a warning at startup for any of these that are missing in production (see `src/instrumentation.ts`).

## Project structure

```
src/
├── app/
│   ├── page.tsx                     # landing
│   ├── f/[slug]/                    # responder (FormSession = the voice engine)
│   ├── invite/[token]/              # team-invite acceptance (sign in / create account)
│   ├── admin/                       # dashboard, create, forms/[id] (+ /edit), settings
│   ├── api/{converse,transcribe,tts,create-form}/   # AI + voice routes
│   └── auth/{callback,signout}/
├── proxy.ts                         # /admin gate (allowlist + team membership)
├── lib/
│   ├── llm.ts                       # Groq → Cerebras chain
│   ├── branching.ts                 # deterministic decision-tree routing
│   ├── authz.ts                     # per-form roles (owner / moderator / viewer)
│   ├── api-errors.ts                # typed error codes + UI mapping
│   ├── hooks/{useTTS,useVoiceRecorder}.ts
│   └── actions/                     # server actions (forms, submit, members, sessions, …)
└── components/{admin,landing,form,voice}/
supabase/{schema.sql, migrations/}
```
